# Fully synthetic validation challenge

All structural fields, material parameters, loading protocols, observations,
noise, and hidden test targets are generated by the repository itself.

## Purpose

The synthetic challenge tests the full computational workflow against known
latent parameters and held-out loading protocols:

1. generate a hidden two-family material and architecture;
2. produce noisy training observations;
3. provide disjoint held-out loading protocols;
4. fit parameters using training data only;
5. predict the hidden test stresses;
6. score generalization and uncertainty;
7. repeat the workflow across noise realizations and architecture perturbations.

## Training protocols

- area-preserving extension along \(x\);
- area-preserving extension along \(y\);
- simple shear;
- small isotropic dilation.

## Held-out protocols

- off-axis extension;
- combined extension and shear;
- axial compression;
- transverse shear.

The protocol names are disjoint by construction. Test deformation gradients
and requested stress components are public, while test stress targets are
hidden from the fitting function.

## Metrics

The validation report includes:

- RMSE;
- MAE;
- normalized RMSE;
- coefficient of determination \(R^2\);
- noise-weighted RMSE;
- fraction of errors within two assumed noise standard deviations;
- maximum absolute error;
- protocol-wise scores.

## Interpretation

This stage is computational verification plus synthetic predictive validation.
It demonstrates that the implemented inverse workflow can recover and
generalize within a controlled data-generating system. Its conclusions are restricted to the documented synthetic data-generating
process.
